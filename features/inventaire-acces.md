# Inventaire et acces

## Objectif

Suivre le materiel, les comptes, les licences, les maillots, les assets et les acces confies aux membres ou staff.

## Ce que la feature doit permettre

- Ajouter un element d'inventaire.
- Assigner un element a un membre.
- Suivre les retours de materiel.
- Garder les licences ou comptes confies.
- Noter les acces importants.
- Lier l'inventaire a l'onboarding ou au depart.
- Archiver un element qui n'est plus utilise.

## Donnees a garder

- Nom de l'element.
- Type.
- Etat.
- Responsable.
- Membre assigne.
- Date d'attribution.
- Date de retour prevue.
- Date de retour effective.
- Notes internes.
- Documents ou justificatifs.

## Types possibles

- Materiel.
- Maillot.
- Compte.
- Licence.
- Acces logiciel.
- Asset graphique.
- Peripherique.
- Autre.

## Permissions

- Voir l'inventaire.
- Ajouter un element.
- Modifier un element.
- Assigner un element.
- Marquer un retour.
- Archiver un element.

## A clarifier

- Si les mots de passe doivent etre stockes ailleurs et jamais dans cette feature.
- Si les membres peuvent voir ce qui leur est assigne.
- Si le retour materiel doit etre obligatoire au depart d'un membre.
